# Button docs

## Look ma - **NO JS**!

<section style={{width: 600}}>
I'm a big fan of simple things. If you don't need something don't use it.

That's why in this case I decided to implement button component only with **CSS**.

In this example implementation you can make your button with any tag you want. Moreover this implementation 
can be used for any library/framework. No extra runtime.

I also followed BEM approach. I chose it only for simplicity and readability, however I'd prefer semantic
approach (like in **SemanticUI** and other CSS frameworks), because BEM classes are huge.
</section>


## Button

### Required params

Button has four possible kinds:

- primary `btn_primary`
- secondary `btn_secondary`
- flat `btn_flat`
- destructive `btn_destructive`

Button also has the following sizes:

- L `btn_size_l`
- S `btn_size_s`

### Optional params

Button can have icons (from left - **prefix** (`btn_with_prefix`) and from right - **postfix** (`btn_with_postfix`)).

If the button should have loading state don't forget to add .loading class (`loading`) and `btn-loader` component inside.  

### Button elements

Button has the following elements which can be used inside:

- button text (`btn-text`) (This thing can be renamed to btn-content, it depends on the component requiremenets)
- button icon (`btn-icon`)
- button loader (`btn-loader`)

If you render just text you don't need to worry about these elements. However if you need to render an icon inside your button
or a loader, then button text comes in handy. I'll show the examples below.

### Caveats

I didn't support icons from both sides (maybe it's not needed).

## How to make buttons

<section style={{width: 600}}>

The button should always have the size and the kind. You need to start with classes `int btn`

`int` - is the scope of intelas design system

`btn` - is the name of the button component

</section>

Below I'll consider only large size primary buttons.

### Primary large button

`<button class="int btn btn_primary btn_size_l">Hi!</button>`

You see - very easy!

<button class="int btn btn_primary btn_size_l">Hi!</button>

### Icon with loader

Loader - is the icon in this case :)

`btn-loader` element should be **before** `btn-text`.

Button in `loading` state will preserve initial width (which depends on text content).

```
<button class="int btn btn_primary btn_size_l loading">
    <Icon class="btn-loader" name="info"/>
    <span class="btn-text">Some really long text</span>
</button>
```
